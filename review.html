Logic overview

The key is to use a Linear scale question with per-answer navigation. Answers 1–3 go to the end; answers 4–5 go to a section holding the follow-up question.

Step 1
Build your rating question
  • Create a new question with type “Linear scale”.
  • Set the range to 1–5 (or 1–10 if you prefer).
  • Label low and high ends (e.g. “Very unlikely” → “Very likely”).
Step 2
Add a follow-up section
  • Insert a new section below the rating question.
  • Place your open text follow-up question in this section.
  • This section is only for “4–5” respondents.
Step 3
Configure branching
  • On the rating question, open the ⋮ menu.
  • Select “Go to section based on answer”.
  • Set 1–3 → next (or submit), 4–5 → follow-up section.

How to set this up

Quick checklist from Google Forms to GitHub Pages, so you can reproduce this end-to-end without extra tooling or build steps.

Google Forms
Configure the survey
  • Create the rating and follow-up sections as described above.
  • Enable “Go to section based on answer” on the linear scale.
  • Optionally collect email addresses and restrict domain.
Embed
Get the iframe code
  • Click “Send” in Google Forms.
  • Choose the <> embed option.
  • Copy the src="…" URL and paste it into this file.
GitHub Pages
Publish the page
  • Create a repo named username.github.io (or any repo).
  • Add this file as index.html in the root.
  • Enable GitHub Pages for the repo (branch: main / docs).
Static front-end only. All survey data is collected and stored by Google Forms, not by GitHub Pages.
